📋 Campaign Management API

Complete REST API for managing advertising campaigns, clients, and notifications

Version: 1.0.0 | Base URL: https://advertising-api.vercel.app | Status: ✅ Live

🔐 Authorization

All endpoints require JWT Bearer token authentication

🚀 Quick Test

Test the API with a simple request

📡 Available Endpoints

GET /
Get API information and available endpoints
curl https://advertising-api.vercel.app/
200 OK - Returns API metadata
GET /health
Health check - verify server is running
curl https://advertising-api.vercel.app/health
200 OK - Server is healthy
GET /swagger.json
Get OpenAPI 3.0 specification
curl https://advertising-api.vercel.app/swagger.json
200 OK - Returns OpenAPI spec
POST /auth/login
Authenticate and get JWT token
curl -X POST https://advertising-api.vercel.app/auth/login \ -H "Content-Type: application/json" \ -d '{"username":"admin","password":"password"}'
200 OK - Returns JWT token
GET /campaigns
List all advertising campaigns
curl https://advertising-api.vercel.app/campaigns \ -H "Authorization: Bearer YOUR_TOKEN"
200 OK - Returns campaigns array
POST /campaigns
Create new campaign
curl -X POST https://advertising-api.vercel.app/campaigns \ -H "Authorization: Bearer YOUR_TOKEN" \ -H "Content-Type: application/json" \ -d '{"name":"Q4 Campaign","status":"active"}'
201 Created
GET /clients
List all clients
curl https://advertising-api.vercel.app/clients \ -H "Authorization: Bearer YOUR_TOKEN"
200 OK - Returns clients array
GET /notifications
Get real-time notifications
curl https://advertising-api.vercel.app/notifications \ -H "Authorization: Bearer YOUR_TOKEN"
200 OK - Returns notifications
